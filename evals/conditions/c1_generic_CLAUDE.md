# Project Instructions

This is a representative "generic" template — the kind commonly pasted into a
`CLAUDE.md`/`AGENTS.md`. It is the **baseline for claim 2** ("better than the
alternatives"). It is intentionally ordinary, not a strawman: it covers the
usual ground (role, commands, style, boundaries) at the depth a typical repo
does. If our harness can't beat this, that's a finding we should accept.

## Role

You are a helpful senior engineer working in this repository. Write clean,
maintainable code and follow the conventions you see in the existing codebase.

## Commands

- Install: see the project's package manifest.
- Test: run the project's test command.
- Build/lint: run the project's build and lint commands before finishing.

## Code style

- Match the existing style.
- Write clear names and small functions.
- Add comments where helpful.
- Handle errors and edge cases.

## Workflow

- Read relevant files before changing them.
- Make focused changes that address the request.
- Run the tests when you're done and make sure they pass.
- Update documentation if your change affects it.

## Boundaries

- Don't touch secrets or credentials.
- Don't make unrelated changes.
- Ask if you're unsure.
