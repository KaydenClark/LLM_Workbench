# Prototype

State the question, the observable verdict, and the disposal boundary first.
Choose the smallest artifact that makes the uncertainty visible: a runnable
logic loop for state behavior or a few contrasting UI variants for interaction
and appearance. Follow local conventions and expose relevant state.

Keep it isolated, clearly temporary, and easy to run. Capture the validated
decision and any evidence in the owning spec, then remove the throwaway work
or retain it only when that spec explicitly needs it as proof.
